# Задание: Реализация системы аутентификации и профиля пользователя на PHP

## Цель задания

Изучить базовые принципы работы PHP-приложения с базой данных, реализовать аутентификацию пользователя, отображение персональной информации и возможность редактирования профиля.

## Описание задания

Необходимо доработать существующий пример страницы входа в систему на PHP. В текущей версии логин и пароль проверяются через заранее прописанные, hard-coded данные. Требуется заменить эту логику на проверку учетных данных через базу данных.

## Функциональные требования

### 1. Вход пользователя в систему
- Пользователь вводит логин и пароль
- Данные проверяются не в коде, а в базе данных
- При успешной аутентификации пользователь перенаправляется на dashboard

### 2. Dashboard
- После входа пользователь попадает на страницу dashboard
- На dashboard отображается информация о текущем пользователе
- Пользователь видит своё имя

### 3. Профиль пользователя
- Пользователь может нажать на своё имя
- Открывается страница редактирования профиля
- Пользователь может изменить базовую информацию: имя, email и другие простые поля

### 4. Выход из системы
- Пользователь может нажать кнопку logout
- После выхода пользователь возвращается на страницу входа
- Доступ к dashboard без аутентификации запрещён

## Технические требования

- Использовать PHP
- Использовать базу данных для хранения пользователей
- Не использовать hard-coded логины и пароли
- Реализовать работу с сессиями PHP
- Защитить dashboard и страницу профиля от неавторизованного доступа
- Код должен быть простым, понятным и подходящим для учебного проекта

## Ожидаемый результат

Простой PHP-проект в котором пользователь может:

1. Войти в систему через данные из базы данных
2. Попасть на dashboard
3. Увидеть информацию о себе
4. Перейти к редактированию своего профиля
5. Сохранить изменения
6. Выйти из системы

## Основная учебная цель

Практическое изучение связки:

**PHP + Database + Sessions + Login System + User Profile Editing**
